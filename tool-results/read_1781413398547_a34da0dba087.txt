     1→     1→# VS Code Web Replica
     2→     2→
     3→     3→A full-featured, browser-based VS Code IDE replica built with Next.js 16, featuring a polyglot microservices backend — TypeScript (Core API), Go (Sandbox), Rust (Search), and Python (AI Copilot). Designed to deliver a production-grade, code-server-like development experience directly in the browser with full responsive support for mobile, tablet, and desktop devices.

    38→    38→## Features
    39→    39→
    40→    40→### Core IDE Experience
    41→    41→- **Full VS Code Layout** — Title bar, activity bar, sidebar, editor area, bottom panel, and status bar faithfully recreated
    42→    42→- **Monaco Code Editor** — Syntax highlighting for 30+ languages, split editor (up to 3 panes), minimap, breadcrumbs, bracket matching
    43→    43→- **5 Color Themes** — Dark+ (default), Light+, Monokai, Solarized Dark, and GitHub themes with 35+ CSS custom properties each
    44→    44→- **Virtual File System** — In-memory file tree with CRUD operations, context menus, drag-and-drop, rename, expand/collapse
    45→    45→- **Command Palette** — `cmdk`-powered palette with command mode (`>` prefix) and quick file open mode (`Ctrl+P`)
    46→    46→- **Settings Panel** — Full settings UI with categories, search, and persistence to localStorage
    47→    47→
    48→    48→### Terminal & Execution
    49→    49→- **Simulated Terminal** — 40+ Unix commands (ls, cd, git, npm, cat, grep, find, etc.), tab completion, command history, multi-tab support
    50→    50→- **Code Execution Sandbox** — Docker-based code execution with resource limits, network isolation, and SSE streaming output
    51→    51→- **Supported Languages** — JavaScript, TypeScript, Python, Go, Rust, and more with local fallback execution
    52→    52→
    53→    53→### AI & Intelligence
    54→    54→- **AI Code Completion (Copilot)** — Inline code suggestions powered by a Python copilot service with template-based fallback
    55→    55→- **Streaming Completions** — Server-Sent Events (SSE) for real-time completion streaming
    56→    56→- **Context-Aware** — Language-specific templates and context extraction for intelligent suggestions
    57→    57→- **Tab to Accept / Esc to Dismiss** — VS Code-style inline completion interaction
    58→    58→
    59→    59→### Collaboration & Real-Time
    60→    60→- **WebSocket Communication** — Socket.IO for real-time terminal I/O, file watching, LSP relay, and AI completion streaming
    61→    61→- **Service Health Monitoring** — Real-time dashboard showing health status of all 4 backend microservices
    62→    62→- **Port Forwarding** — Start, stop, and manage forwarded ports directly from the UI
    63→    63→
    64→    64→### Developer Tools
    65→    65→- **Source Control (Git)** — Stage/unstage changes, commit, branch switching, diff indicators (simulated)
    66→    66→- **Extension Marketplace** — Browse, install, uninstall, and toggle extensions
    67→    67→- **Full-Text Search** — Rust-powered search with inverted index, trie-based filename autocomplete, regex and fuzzy matching
    68→    68→- **Notification Center** — Toast and persistent notifications with actionable buttons
    69→    69→- **Workspace Trust Dialog** — VS Code-style workspace trust flow
    70→    70→
    71→    71→### Mobile & Responsive
    72→    72→- **Touch-Optimized** — 44px minimum touch targets, swipe gestures, long-press actions
    73→    73→- **Adaptive Layout** — Mobile (overlay sidebar, bottom nav), Tablet (compact sidebar), Desktop (full layout)
    74→    74→- **Landscape & Foldable** — Special handling for landscape orientation and foldable devices
    75→    75→- **Virtual Keyboard** — Proper `100dvh`/`100svh` handling for mobile keyboard interactions
    76→    76→- **Safe Area Insets** — Support for notched devices with `env(safe-area-inset-*)`
    77→    77→
    78→    78→### Security & Auth
    79→    79→- **Authentication** — code-server-style login with password, rate limiting, and local fallback
    80→    80→- **Sandbox Isolation** — Docker containers with memory limits, CPU quotas, PID limits, and disabled networking
    81→    81→- **Workspace Trust** — Explicit trust dialog before executing code in untrusted workspaces

    85→    85→## Architecture Overview
    86→    86→
    87→    87→```
    88→    88→                          ┌─────────────────────────────────────────────────────┐
    89→    89→                          │                    Caddy (Port 81)                   │
    90→    90→                          │              Reverse Proxy + TLS Termination         │
    91→    91→                          └──────────┬──────────┬──────────┬──────────┬─────────┘
    92→    92→                                     │          │          │          │
    93→    93→                          ┌──────────▼──┐ ┌─────▼────┐ ┌───▼────┐ ┌──▼─────────┐
    94→    94→                          │  Frontend   │ │ Core API │ │Search  │ │  Copilot   │
    95→    95→                          │  Next.js    │ │  Node.js │ │ Rust   │ │  Python    │
    96→    96→                          │  Port 3000  │ │ Port 3001│ │Port 3003│ │ Port 3004  │
    97→    97→                          └─────────────┘ └────┬─────┘ └────────┘ └────────────┘
    98→    98→                                               │
    99→    99→                                          ┌────▼─────┐
   100→   100→                                          │ Sandbox  │
   101→   101→                                          │   Go     │
   102→   102→                                          │Port 3002 │
   103→   103→                                          └──────────┘
   104→   104→```
   105→   105→
   106→   106→The application follows a **microservices architecture** with a polyglot backend:
   107→   107→
   108→   108→| Service | Language | Port | Responsibility |
   109→   109→|---------|----------|------|----------------|
   110→   110→| **Frontend** | TypeScript (Next.js) | 3000 | UI rendering, client-side state, API gateway |
   111→   111→| **Core API** | TypeScript (Node.js) | 3001 | REST API, WebSocket routing, service registry |
   112→   112→| **Sandbox** | Go | 3002 | Code execution, Docker container management |
   113→   113→| **Search** | Rust | 3003 | Full-text indexing, filename autocomplete, fuzzy search |
   114→   114→| **Copilot** | Python | 3004 | AI code completion, template engine, context extraction |
   115→   115→| **Caddy** | Go | 81 | Reverse proxy, TLS, port-based routing |

   154→   154→## Getting Started
   155→   155→
   156→   156→### Prerequisites
   157→   157→
   158→   158→- **Node.js** >= 18 (or Bun >= 1.0)
   159→   159→- **Go** >= 1.21 (for Sandbox service)
   160→   160→- **Rust** >= 1.70 (for Search service)
   161→   161→- **Python** >= 3.11 (for Copilot service)
   162→   162→- **Docker** >= 24.0 (for containerized execution & deployment)
   163→   163→- **Docker Compose** >= 2.20
   164→   164→
   165→   165→### Quick Start (Docker)
   166→   166→
   167→   167→The fastest way to get the entire stack running:
   168→   168→
   169→   169→```bash
   170→   170→# Clone the repository
   171→   171→git clone https://github.com/gamingextra/VS-Code-Web-Replica.git
   172→   172→cd VS-Code-Web-Replica
   173→   173→
   174→   174→# Switch to the feature branch
   175→   175→git checkout feature/code-server-enhancements
   176→   176→
   177→   177→# Build and start all services
   178→   178→docker compose up --build
   179→   179→
   180→   180→# Access the application
   181→   181→# Frontend:    http://localhost:81
   182→   182→# Core API:    http://localhost:81?XTransformPort=3001
   183→   183→# Sandbox:     http://localhost:81?XTransformPort=3002
   184→   184→# Search:      http://localhost:81?XTransformPort=3003
   185→   185→# Copilot:     http://localhost:81?XTransformPort=3004
   186→   186→```
   187→   187→
   188→   188→Default login credentials: `password: vscode`
   189→   189→
   190→   190→### Local Development
   191→   191→
   192→   192→For active development, run the frontend and backend services separately:
   193→   193→
   194→   194→```bash
   195→   195→# 1. Install frontend dependencies
   196→   196→bun install
   197→   197→
   198→   198→# 2. Start the Next.js development server
   199→   199→bun dev
   200→   200→
   201→   201→# 3. Start all backend services (from project root)
   202→   202→chmod +x start-backend.sh
   203→   203→./start-backend.sh
   204→   204→```
   205→   205→
   206→   206→The `start-backend.sh` script automatically:
   207→   207→- Detects installed runtimes (Go, Rust, Python)
   208→   208→- Starts each service in the background
   209→   209→- Performs health checks with retries
   210→   210→- Provides graceful shutdown on Ctrl+C
   211→   211→
   212→   212→#### Starting Backend Services Individually
   213→   213→
   214→   214→**Core API (TypeScript/Node.js)**
   215→   215→```bash
   216→   216→cd mini-services/core-api
   217→   217→bun install
   218→   218→bun run index.ts
   219→   219→# Runs on http://localhost:3001
   220→   220→```
   221→   221→
   222→   222→**Sandbox (Go)**
   223→   223→```bash
   224→   224→cd backend/sandbox
   225→   225→go run .
   226→   226→# Runs on http://localhost:3002
   227→   227→# Requires Docker daemon for container execution
   228→   228→```
   229→   229→
   230→   230→**Search (Rust)**
   231→   231→```bash
   232→   232→cd backend/search
   233→   233→cargo run --release
   234→   234→# Runs on http://localhost:3003
   235→   235→```
   236→   236→
   237→   237→**Copilot (Python)**
   238→   238→```bash
   239→   239→cd backend/copilot
   240→   240→pip install -r requirements.txt
   241→   241→python main.py
   242→   242→# Runs on http://localhost:3004
   243→   243→```
   244→   244→
   245→   245→### Backend Services
   246→   246→
   247→   247→Each backend service operates independently with its own health endpoint and graceful degradation. If a backend service is unavailable, the frontend falls back to client-side simulation:
   248→   248→
   249→   249→| Service Unavailable | Fallback Behavior |
   250→   250→|---------------------|-------------------|
   251→   251→| Core API | Local in-memory state, simulated responses |
   252→   252→| Sandbox | Local JavaScript `eval()` for JS, template execution for Python/Go/Rust |
   253→   253→| Search | Client-side in-memory string matching |
   254→   254→| Copilot | Template-based code completion (language-specific snippets) |

   473→   473→### Sandbox Service (Go)
   474→   474→
   475→   475→**Port:** 3002 | **Location:** `backend/sandbox/`
   476→   476→
   477→   477→Docker-based code execution sandbox with strict resource limits and network isolation.

   483→   483→- Support for JavaScript, TypeScript, Python, Go, Rust, and more
   484→   484→- Graceful shutdown with 5-second timeout

   489→   489→|----------|-------|
   490→   490→| Memory | 512 MB |
   491→   491→| CPU Quota | 100,000 (1 CPU) |
   492→   492→| PIDs | 100 |
   493→   493→| Network | Disabled |
   494→   494→| Timeout | 30 seconds |

   496→   496→**Environment Variables:**

   508→   508→### Search Service (Rust)
   509→   509→
   510→   510→**Port:** 3003 | **Location:** `backend/search/`
   511→   511→
   512→   512→High-performance file search and indexing engine built with Axum and Tokio.
   513→   513→
   514→   514→**Key Features:**
   515→   515→- **Inverted Index** — Full-text content search with term frequency ranking
   516→   516→- **Trie Data Structure** — O(k) filename autocomplete where k is the query length
   517→   517→- **File Watching** — Incremental reindexing via the `notify` crate (debounced 500ms)
   518→   518→- **Regex Search** — Full regular expression support for advanced queries
   519→   519→- **Fuzzy Matching** — Approximate string matching for typo-tolerant search
   520→   520→- **Structured Logging** — JSON-formatted logs with the `tracing` crate
   521→   521→
   522→   522→**API Endpoints:**
   523→   523→
   524→   524→| Method | Endpoint | Description |
   525→   525→|--------|----------|-------------|
   526→   526→| POST | `/api/search` | Full-text content search |
   527→   527→| POST | `/api/search/filenames` | Filename search with trie autocomplete |
   528→   528→| GET | `/api/search/suggest` | Search suggestions |
   529→   529→| POST | `/api/index` | Trigger manual reindex |
   530→   530→| GET | `/api/index/stats` | Index statistics |
   531→   531→| GET | `/health` | Health check |

   535→   535→### Copilot Service (Python)
   536→   536→
   537→   537→**Port:** 3004 | **Location:** `backend/copilot/`
   538→   538→
   539→   539→AI-powered code completion engine built with FastAPI.
   540→   540→
   541→   541→**Key Features:**
   542→   542→- **Synchronous Completions** — Single-request code completion with confidence scoring
   543→   543→- **Streaming Completions** — SSE-based token-by-token streaming for real-time suggestions
   544→   544→- **Context-Aware Engine** — Extracts surrounding code context for better suggestions
   545→   545→- **Language Templates** — Pre-built completion templates for TypeScript, JavaScript, Python, Go, and Rust
   546→   546→- **Confidence Scoring** — Each completion includes a confidence level (high/medium/low)
   547→   547→
   548→   548→**API Endpoints:**
   549→   549→
   550→   550→| Method | Endpoint | Description |
   551→   551→|--------|----------|-------------|
   552→   552→| POST | `/api/completions` | Synchronous code completion |
   553→   553→| POST | `/api/completions/stream` | Streaming code completion (SSE) |
   554→   554→| GET | `/api/completions/health` | Completion engine health |
   555→   555→| GET | `/health` | Service health check |
   556→   556→
   557→   557→**Request Format:**
   558→   558→```json
   559→   559→{
   560→   560→  "file_path": "src/index.ts",
   561→   561→  "language": "typescript",
   562→   562→  "prefix": "function add(a: number, b: number) ",
   563→   563→  "suffix": "\n}",
   564→   564→  "cursor_position": 35
   565→   565→}
   566→   566→```
   567→   567→
   568→   568→**Response Format:**
   569→   569→```json
   570→   570→{
   571→   571→  "completions": [
   572→   572→    {
   573→   573→      "text": "{\n  return a + b;\n",

   575→   575→      "confidence": 0.85,
   576→   576→      "source": "template"
   577→   577→    }
   578→   578→  ]
   579→   579→}
   580→   580→```

   585→   585→
   586→   586→### Client-Side API Gateway
   587→   587→
   588→   588→The `src/lib/api-client.ts` module provides typed functions for all backend endpoints. It implements a **cascading fallback** strategy:
   589→   589→
   590→   590→```
   591→   591→1. Try direct service call (e.g., localhost:3003/api/search)
   592→   592→2. If failed → Try Core API proxy (e.g., localhost:3001/api/search)
   593→   593→3. If failed → Use local client-side simulation
   594→   594→```
   595→   595→
   596→   596→The `XTransformPort` query parameter enables Caddy-based port routing in production.
   597→   597→
   598→   598→**Available Functions:**
   599→   599→
   600→   600→```typescript
   601→   601→// Health & Services
   602→   602→apiClient.healthCheck()
   603→   603→apiClient.getServices()
   604→   604→
   605→   605→// Workspaces
   606→   606→apiClient.getWorkspaces()
   607→   607→apiClient.createWorkspace(data)
   608→   608→
   609→   609→// Files
   610→   610→apiClient.getFiles(workspaceId)
   611→   611→apiClient.createFile(data)
   612→   612→apiClient.updateFile(id, data)
   613→   613→apiClient.deleteFile(id)
   614→   614→
   615→   615→// Search
   616→   616→apiClient.searchFiles(query, options?)
   617→   617→
   618→   618→// Execution
   619→   619→apiClient.executeCode(code, language, options?)
   620→   620→
   621→   621→// AI Completions
   622→   622→apiClient.getCompletions(request)
   623→   623→
   624→   624→// Authentication
   625→   625→apiClient.login(username, password)
   626→   626→
   627→   627→// Git

   629→   629→
