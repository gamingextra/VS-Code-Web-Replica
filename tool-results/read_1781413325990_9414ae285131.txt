     1вҶ’# VS Code Web Replica
     2вҶ’
     3вҶ’A full-featured, browser-based VS Code IDE replica built with Next.js 16, featuring a polyglot microservices backend вҖ” TypeScript (Core API), Go (Sandbox), Rust (Search), and Python (AI Copilot). Designed to deliver a production-grade, code-server-like development experience directly in the browser with full responsive support for mobile, tablet, and desktop devices.

    38вҶ’## Features
    39вҶ’
    40вҶ’### Core IDE Experience
    41вҶ’- **Full VS Code Layout** вҖ” Title bar, activity bar, sidebar, editor area, bottom panel, and status bar faithfully recreated
    42вҶ’- **Monaco Code Editor** вҖ” Syntax highlighting for 30+ languages, split editor (up to 3 panes), minimap, breadcrumbs, bracket matching
    43вҶ’- **5 Color Themes** вҖ” Dark+ (default), Light+, Monokai, Solarized Dark, and GitHub themes with 35+ CSS custom properties each
    44вҶ’- **Virtual File System** вҖ” In-memory file tree with CRUD operations, context menus, drag-and-drop, rename, expand/collapse
    45вҶ’- **Command Palette** вҖ” `cmdk`-powered palette with command mode (`>` prefix) and quick file open mode (`Ctrl+P`)
    46вҶ’- **Settings Panel** вҖ” Full settings UI with categories, search, and persistence to localStorage
    47вҶ’
    48вҶ’### Terminal & Execution
    49вҶ’- **Simulated Terminal** вҖ” 40+ Unix commands (ls, cd, git, npm, cat, grep, find, etc.), tab completion, command history, multi-tab support
    50вҶ’- **Code Execution Sandbox** вҖ” Docker-based code execution with resource limits, network isolation, and SSE streaming output
    51вҶ’- **Supported Languages** вҖ” JavaScript, TypeScript, Python, Go, Rust, and more with local fallback execution
    52вҶ’
    53вҶ’### AI & Intelligence
    54вҶ’- **AI Code Completion (Copilot)** вҖ” Inline code suggestions powered by a Python copilot service with template-based fallback
    55вҶ’- **Streaming Completions** вҖ” Server-Sent Events (SSE) for real-time completion streaming
    56вҶ’- **Context-Aware** вҖ” Language-specific templates and context extraction for intelligent suggestions
    57вҶ’- **Tab to Accept / Esc to Dismiss** вҖ” VS Code-style inline completion interaction
    58вҶ’
    59вҶ’### Collaboration & Real-Time
    60вҶ’- **WebSocket Communication** вҖ” Socket.IO for real-time terminal I/O, file watching, LSP relay, and AI completion streaming
    61вҶ’- **Service Health Monitoring** вҖ” Real-time dashboard showing health status of all 4 backend microservices
    62вҶ’- **Port Forwarding** вҖ” Start, stop, and manage forwarded ports directly from the UI
    63вҶ’
    64вҶ’### Developer Tools
    65вҶ’- **Source Control (Git)** вҖ” Stage/unstage changes, commit, branch switching, diff indicators (simulated)
    66вҶ’- **Extension Marketplace** вҖ” Browse, install, uninstall, and toggle extensions
    67вҶ’- **Full-Text Search** вҖ” Rust-powered search with inverted index, trie-based filename autocomplete, regex and fuzzy matching
    68вҶ’- **Notification Center** вҖ” Toast and persistent notifications with actionable buttons
    69вҶ’- **Workspace Trust Dialog** вҖ” VS Code-style workspace trust flow
    70вҶ’
    71вҶ’### Mobile & Responsive
    72вҶ’- **Touch-Optimized** вҖ” 44px minimum touch targets, swipe gestures, long-press actions
    73вҶ’- **Adaptive Layout** вҖ” Mobile (overlay sidebar, bottom nav), Tablet (compact sidebar), Desktop (full layout)
    74вҶ’- **Landscape & Foldable** вҖ” Special handling for landscape orientation and foldable devices
    75вҶ’- **Virtual Keyboard** вҖ” Proper `100dvh`/`100svh` handling for mobile keyboard interactions
    76вҶ’- **Safe Area Insets** вҖ” Support for notched devices with `env(safe-area-inset-*)`
    77вҶ’
    78вҶ’### Security & Auth
    79вҶ’- **Authentication** вҖ” code-server-style login with password, rate limiting, and local fallback
    80вҶ’- **Sandbox Isolation** вҖ” Docker containers with memory limits, CPU quotas, PID limits, and disabled networking
    81вҶ’- **Workspace Trust** вҖ” Explicit trust dialog before executing code in untrusted workspaces

   106вҶ’The application follows a **microservices architecture** with a polyglot backend:
   107вҶ’
   108вҶ’| Service | Language | Port | Responsibility |

   110вҶ’| **Frontend** | TypeScript (Next.js) | 3000 | UI rendering, client-side state, API gateway |
   111вҶ’| **Core API** | TypeScript (Node.js) | 3001 | REST API, WebSocket routing, service registry |
   112вҶ’| **Sandbox** | Go | 3002 | Code execution, Docker container management |
   113вҶ’| **Search** | Rust | 3003 | Full-text indexing, filename autocomplete, fuzzy search |
   114вҶ’| **Copilot** | Python | 3004 | AI code completion, template engine, context extraction |
   115вҶ’| **Caddy** | Go | 81 | Reverse proxy, TLS, port-based routing |

   154вҶ’## Getting Started
   155вҶ’
   156вҶ’### Prerequisites
   157вҶ’
   158вҶ’- **Node.js** >= 18 (or Bun >= 1.0)
   159вҶ’- **Go** >= 1.21 (for Sandbox service)
   160вҶ’- **Rust** >= 1.70 (for Search service)
   161вҶ’- **Python** >= 3.11 (for Copilot service)
   162вҶ’- **Docker** >= 24.0 (for containerized execution & deployment)
   163вҶ’- **Docker Compose** >= 2.20
   164вҶ’
   165вҶ’### Quick Start (Docker)
   166вҶ’
   167вҶ’The fastest way to get the entire stack running:
   168вҶ’
   169вҶ’```bash
   170вҶ’# Clone the repository
   171вҶ’git clone https://github.com/gamingextra/VS-Code-Web-Replica.git
   172вҶ’cd VS-Code-Web-Replica
   173вҶ’
   174вҶ’# Switch to the feature branch
   175вҶ’git checkout feature/code-server-enhancements
   176вҶ’
   177вҶ’# Build and start all services
   178вҶ’docker compose up --build
   179вҶ’
   180вҶ’# Access the application
   181вҶ’# Frontend:    http://localhost:81
   182вҶ’# Core API:    http://localhost:81?XTransformPort=3001
   183вҶ’# Sandbox:     http://localhost:81?XTransformPort=3002
   184вҶ’# Search:      http://localhost:81?XTransformPort=3003
   185вҶ’# Copilot:     http://localhost:81?XTransformPort=3004
   186вҶ’```
   187вҶ’
   188вҶ’Default login credentials: `password: vscode`
   189вҶ’
   190вҶ’### Local Development
   191вҶ’
   192вҶ’For active development, run the frontend and backend services separately:
   193вҶ’
   194вҶ’```bash
   195вҶ’# 1. Install frontend dependencies
   196вҶ’bun install
   197вҶ’
   198вҶ’# 2. Start the Next.js development server
   199вҶ’bun dev
   200вҶ’
   201вҶ’# 3. Start all backend services (from project root)
   202вҶ’chmod +x start-backend.sh
   203вҶ’./start-backend.sh
   204вҶ’```
   205вҶ’
   206вҶ’The `start-backend.sh` script automatically:
   207вҶ’- Detects installed runtimes (Go, Rust, Python)
   208вҶ’- Starts each service in the background
   209вҶ’- Performs health checks with retries
   210вҶ’- Provides graceful shutdown on Ctrl+C
   211вҶ’
   212вҶ’#### Starting Backend Services Individually
   213вҶ’
   214вҶ’**Core API (TypeScript/Node.js)**
   215вҶ’```bash
   216вҶ’cd mini-services/core-api
   217вҶ’bun install
   218вҶ’bun run index.ts
   219вҶ’# Runs on http://localhost:3001
   220вҶ’```
   221вҶ’
   222вҶ’**Sandbox (Go)**
   223вҶ’```bash
   224вҶ’cd backend/sandbox
   225вҶ’go run .
   226вҶ’# Runs on http://localhost:3002
   227вҶ’# Requires Docker daemon for container execution
   228вҶ’```
   229вҶ’
   230вҶ’**Search (Rust)**
   231вҶ’```bash
   232вҶ’cd backend/search
   233вҶ’cargo run --release
   234вҶ’# Runs on http://localhost:3003
   235вҶ’```
   236вҶ’
   237вҶ’**Copilot (Python)**
   238вҶ’```bash
   239вҶ’cd backend/copilot
   240вҶ’pip install -r requirements.txt
   241вҶ’python main.py
   242вҶ’# Runs on http://localhost:3004
   243вҶ’```
   244вҶ’
   245вҶ’### Backend Services
   246вҶ’
   247вҶ’Each backend service operates independently with its own health endpoint and graceful degradation. If a backend service is unavailable, the frontend falls back to client-side simulation:
   248вҶ’
   249вҶ’| Service Unavailable | Fallback Behavior |
   250вҶ’|---------------------|-------------------|
   251вҶ’| Core API | Local in-memory state, simulated responses |
   252вҶ’| Sandbox | Local JavaScript `eval()` for JS, template execution for Python/Go/Rust |
   253вҶ’| Search | Client-side in-memory string matching |
   254вҶ’| Copilot | Template-based code completion (language-specific snippets) |

   593вҶ’3. If failed вҶ’ Use local client-side simulation
   594вҶ’```
