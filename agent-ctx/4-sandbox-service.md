# Task 4: Go Code Execution & Sandboxing Service

## Summary
Created the complete Go microservice for secure code execution in sandboxed Docker containers. The service provides an HTTP API on port 3002 with real-time output streaming via Server-Sent Events (SSE).

## Architecture
- **HTTP Server**: Gin framework on port 3002
- **Docker Integration**: Docker SDK for container lifecycle management
- **Security**: Network isolation, read-only filesystem, resource limits, capability dropping, no-new-privileges
- **Streaming**: Real-time output via SSE with multiplexed stdout/stderr

### `/home/z/my-project/backend/sandbox/runner/runner.go`
- Core execution engine with Docker container lifecycle:
  1. Validate language & request
  2. Create temp directory with source code
  3. Pull Docker image if not available
  4. Create container with security constraints
  5. Attach for I/O (stdin/stdout/stderr)
  6. Start container and stream output
  7. Wait for completion or timeout
  8. Cleanup container and temp files
- Real-time output streaming via subscriber channels
- Docker multiplexed stream parsing (8-byte header protocol)
- Timeout enforcement with context cancellation
- In-memory result storage with concurrent access protection

## Integration Notes
- The service runs on port 3002
- Frontend should use `XTransformPort=3002` query parameter for API requests
- Docker socket must be mounted at `/var/run/docker.sock` for the service to function
- The `go.sum` file will be generated by `go mod tidy` during Docker build
- The existing `codeExecutionStore.ts` frontend store uses the same result structure (`id`, `status`, `output`, `error`, `exitCode`, `executionTime`)
